CSE 220c: Week 03

Ponder and Prove

Week 3 Ponder and Prove: Standard Template Library and Iterators

Overview

Task: Upgrade the bank system to handle more than one account.

Purpose: Learn how to use the Standard Template Library and iterators.

Instructions

These instructions are designed to help you break your programming into small pieces to make it easier for you to write and debug. You only turn in the final program, after all of the parts have been written, but it will be easier if you make sure that each part is complete before proceeding to the next.


Part 1.

Add the ability to store more than one account to your bank program. You should use one of the Standard Template Library containers to hold the accounts. You may select which type of container to use.

Your program should no longer automatically create an account when it starts. You will only create a new account when the user selects this as an option. You should create a new menu option that will enable the user to create a new account. Once the new account is added, store the new account in your container. You will likely need to temporarily remove or comment out the code for some of the other menu functions to get this to work. We'll be adding those functions back later.

Example input and output (input is underlined):

Main Menu:
0. Quit Program
1. Display account information
2. Add a deposit to an account
3. Withdraw from an account
4. Add new account
Your choice: 4
Enter the name: George
Enter the initial balance: 100
