The Why of C++

Some Background

Before and during the early 1970's, it was common to write operating systems in assembly, a low-level programming language. The first implementation of the Unix operating system was written in assembly to run on the PDP-7 computer and later was ported, by modifying the original assembly code, to the PDP-11. Since writing and porting assembly for larger systems is difficult, first Ken Thompson and later Dennis Richie decided to create languages to make this process easier. Thompson created the B language that Richie used as a starting point when creating the earliest versions of C. By the end of 1973, the Unix kernel had been rewritten completely in C, making it easier to port to new versions of hardware.

C, then, was a language designed for writing operating systems, device drivers, and other types of systems. The patterns you use when you write C are those you would use if you were writing code for the PDP-11. Thankfully, the portable C compiler led the way to making C available on multiple hardware systems that don't follow the PDP-11 hardware patterns while maintaining the PDP-11 coding patterns. Porting Unix is not an insignificant task. In spite of this, the C language has made several flavors of Unix such as Solaris, MacOS, HP-UX, the BSD variants, and AIX possible. Linux, a Unix-like system, is also written in C.

In the late 1970's, Bjarne Stroustrup was a graduate student. He had been using a language called Simula to simulate parallel data flow in computing systems. Among other things, Bjarne liked the way Simula allowed related functions to be grouped together into classes, and that these classes could inherit properties and functionality from base classes. Unfortunately for Bjarne, the Simula language had drawbacks, as any language does; Simula struggled to scale. It ran too large and too slow for Bjarne's project. Facing the collapse of his project, Bjarne switched languages to BCPL, a language he said was horrible. As to how easy BCPL was to use, Bjarne said, "BCPL makes C look like a very high-level language."

After leaving the university with his PHD, Bjarne continued working on the problem he had faced. He eventually created C with Objects, and then C++. His goal was to create a language that allowed the organizational goodies he liked from Simula, while still being able to scale like C.

If you are interested in a career using C++, or just want to know it in depth, Stroustrup's seminal book, The C++ Programming Language, is available in hardback, paperback and electronically.

Python and C++ are Related

Take a look at Figure 1. It shows the ancestry of both C++ and Python. Since C is an ancestor of both C++ and Python you can expect to see some things in C++ that remind you of what you learned about coding in Python. C is not Python's only ancestor. A very strong contributor to Python is the Modula language. Because of this ancestry, modules exist in Python that are somewhat similar to classes in C++. Since Modula is not an ancestor of C++, C++ has no concept of a range and, as you will see, the syntax and structure of for loops is different from what you learned for Python though the loop concepts remain the same.

For a more complete understanding of the relationship between C and C++ read Stroustrup's paper on the subject, written in 2002. There, students will learn, among other things, that C++ is not a superset of C. The article does not, however, include changes made to both languages since 2002.

Installations

The IDE


Microsoft Visual Studio Community Edition (Windows)

Note: Windows users should install Visual Studio Community, NOT Visual Studio Code. These are two vastly different products. Visual Studio Code is a nice text editor with great extensions, but it is not the right tool for professional C++ development. Visual Studio Community is a full-featured IDE (Integrated Development Environment), including a modern C++ compiler and debugger. Both of these products from Microsoft are free. For this class, make sure you are using Visual Studio Community.

If it isn't already installed on your machine, start by downloading and installing Visual Studio Community Edition. This is NOT the same thing as Visual Studio Code.

Next, watch and follow the steps in the video How to Install Visual Studio 2019 for C++ Programming to see how to complete the download, create your first project, and then run it.

Xcode (MacOS)

If it isn't already installed on your machine, go to the Apple app store download page for Xcode. When you attempt to install, if you don't have the latest version of MacOS, you may see a message saying you don't have a compatible MacOS version. If you see this message, update your OS to the latest version.

Follow the video tutorial Compile C++ Using Xcode to see how to create, compile, and run a C++ project after launching Xcode.

Eclipse (Windows, MacOS, Linux)

Eclipse is a good cross-platform development environment. Eclipse can be downloaded from the website. Note that Eclipse does not come with a compiler, so that will have to be downloaded and installed separately. On Windows, you can download either the original MinGW or the newer MinGW-w64 project. On MacOS, you should use the clang compiler and lldb debugger because they are natively supported (however, as of this writing they may not work well with Eclipse). You will need to install them using Xcode. On Linux, you should just use the native gcc compiler. The gcc compiler should be available to download the normal way you install software for your Linux distribution. This course provides some videos to help you get started with Eclipse.

Workflow

It is important for you to remember that you will want to make a DIFFERENT project for each task you are given. Do NOT create a single project and think you can reuse it for all of your tasks. If you try this, you will regret it. Instead, create a directory into which you can save all of your projects. That way you know where to look when you want to find an old project.

The work you do using these two tools will flow like this. Each time you are asked to do a task, create a directory named for the task. For example, if you were asked for your task to write code that calculated loan payments, you could create a project called loan_pay_calc. Each file you create for this task will be part of this project.

If you follow this pattern for each assignment, you will reduce your workload.

Do NOT create one project for the course and think that you can put the code for all of your tasks in it. If you ignore this advice and do this anyway, your life will be harder, completing your tasks will take longer, and your code will be harder to understand. You don't need these self-inflicted barriers to your learning.

So please, make a separate project for each task!
