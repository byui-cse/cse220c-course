CSE 220C: Week 02

Ponder and Prove

Week 2: Using Classes

Overview

Task: Create a simple bank system to keep track of account balances.

Purpose: Learn how to create C++ classes and use overloaded operators.

Instructions

These instructions are designed to help you break your programming into small pieces to make it easier for you to write and debug. You only turn in the final program, after all of the parts have been written, but it will be easier if you make sure that each part is complete before proceeding to the next.

Before attempting to complete the task, you should read through the following material. These reading links are provided to help you understand what you'll need to do as you complete the task. You don't necessarily need to read every word of each link, but you should browse through each link and spend time on concepts that are not familiar to you.

  • CSE 220C: Defining and Using Classes
  • CSE 220C: C++ Classes Additional Material
  • Learn C++: Class Syntax
  • Learn C++: Method Definitions
  • Learn C++: Operators (Optional reading)
  • Learn C++: Inheritance (Optional reading)

You may need to consult additional documentation not listed here. You should ask for help in our class discussion when you get stuck.

As you write your program code, you will separate your code into different files.

You should create the following files:

  • account.h - Stores the Account class declaration.
  • account.cpp - Stores the Account class implementation.
  • program.cpp - Contains the main function.
You will create header files and cpp source files for each class. You will place the class declaration in a header file, and your class method implementations will go in a cpp source file. Your main function will go in yet another cpp source file. All of your cpp source files will #include the header file for your class. Do not #include cpp files from another cpp file. You should add each source file to the project in your development environment.

Hint: The section called "The Date Class" in the Defining and Using Classes document will be helpful to determine how to separate your code. This document has an example of which things go in the class header file and which things go in the cpp implementation file.

This task will build upon last week's programming task. However, you should still create a new project for this week's task and copy last week's code into your new project. When you submit your program code, submit all of your files together in a single zip file.

Part 1.

Create a simple class that represents a bank account. For this task, you will create a program that has a single account. In later tasks, you will add the ability to store multiple accounts.

The account class will have the following data members:

  • an account ID (integer)
  • account name (string)
  • account balance (float)

All of these members should be private or protected. (This is generally true of all class data members.)

The account class will have a constructor that takes no parameters. This constructor will initialize the data members to empty values.

The account class will have a method that prompts the user to enter the name and the initial balance. The ID of the object will be automatically set by your program with the next ID number available. The first ID number is zero, and subsequent accounts will get numerically increasing ID numbers. A static data member is one way to store the next available ID number (this is separate from the ID data member in each object).

The account class will have a method that will display the account information.

Your main function will create an object of the account class, call the object's input method that asks the user to enter the account information, and call the object's output method to show the account information to the user.

Example input and output (input is underlined):

Enter the name: George
Enter the balance: 100
Account ID: 0  Name: George  Balance: $100.00

Part 5. Optional for extra learning and practice.

As an optional enhancement to your program, implement one or more of the following. These are all optional and are not required to complete this task.

  1. Add an add-onto operator += to the Account class that will add a deposit to the account balance. This operator should take a float as a parameter and return a reference to an Account.

    The declaration of this operator in the header file should look something like this:

    Account& operator+= (float deposit);

    If properly implemented, now you can add a deposit to the account by doing this: 

    my_account += deposit_amount;

  2. Add a subtract-from operator -= to the class that will subtract a withdrawal from the account balance.

    If properly implemented, now you can subtract a withdrawal from the account by doing this: 

    my_account -= withdrawal_amount;

  3. Add a method to the Account class that takes an output stream as a parameter and returns an output stream. The declaration of this method in your header file should look something like this:

    std::ostream& display(std::ostream& out) const;

    You can use the standard object std::cout to pass to this output method.

    If properly implemented, now you can display the account information by passing an output stream to the display method, like this: 

    my_account.display(std::cout);

    This is helpful because we can now provide different output streams, such as a file output stream, rather than being limited to displaying account information only in the console using std::cout

  4. Add an insertion operator (<<) to enable writing the object's data members to an output stream. Because this operator is defined outside the class, it cannot access private or protected members. You will need to use a public method that can be used to write an object's information to an output stream (see optional item number 3 above). Then, you merely need to call it from your insertion operator. Insertion operators take an output stream reference as the first parameter, and a const reference of the class as the second parameter. This operator returns the output stream reference, which is what allows these calls to be chained together. The insertion operator should not be defined as a member of your class; instead, it is defined as a standalone function in the same file as the rest of your class methods. A function prototype for this function should be placed in the class header file below the class definition.

    Modify your main function to use the output operator instead of calling the output method directly, to send the object to std::cout.

    Example:

    Account my_account; // Create an account
my_account.initialize(); // Prompts user for name and account balance

my_account.display(); // Default method to display account information

my_account.display(std::cout); // Using the optional number 3 method

std::cout << my_account; // Do this instead after implementing the << operator
