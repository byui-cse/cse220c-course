CSE 220C: Week 02

Ponder and Prove

Week 2: Using Classes

Overview

Task: Create a simple bank system to keep track of account balances.

Purpose: Learn how to create C++ classes and use overloaded operators.

Instructions

These instructions are designed to help you break your programming into small pieces to make it easier for you to write and debug. You only turn in the final program, after all of the parts have been written, but it will be easier if you make sure that each part is complete before proceeding to the next.

As you write your program code, you will separate your code into different files. You will create header files and cpp source files for each class. You will place the class declaration in a header file, and your class method implementations will go in a cpp source file. Your main function will go in yet another cpp source file. All of your cpp source files will include your header files as needed. This task will build upon last week's programming task. However, you should still create a new project for this week's task and copy last week's code into your new project. When you submit your program code, submit all of your files together in a single zip file.

Part 1.

Create a simple class that represents a bank account. For this task, you will create a program that has a single account. In later tasks, you will add the ability to store multiple accounts.

The account class will have the following data members:

  • an account ID (integer)
  • account name (string)
  • account balance (float)

All of these members should be private or protected. (This is generally true of all class data members.)

The account class will have a constructor that takes no parameters. This constructor will initialize the data members to empty values.

The account class will have a copy constructor that copies all of the data from one instance to another instance. The source instance should be passed by reference.

The account class will have a destructor that takes no parameters and an empty body. We'll add more code here later.

The account class will have a method that prompts the user to enter the name and the initial balance. The ID of the object will be automatically set by your program with the next ID number available. The first ID number is 0, and subsequent accounts will get numerically increasing ID numbers.

The account class will have a method that will send the data members to the output stream passed in as a parameter. The stream should be passed by reference to the method, and should return the same stream reference that is passed to the method.

Your main function will create an object of the account class, call the object's input method that asks the user to enter the account information, and call the object's output method to show the data members to the user. You can use the standard object std::cout to pass to the output method.

Example input and output (input is underlined):

Enter the name: George
Enter the balance: 100
Account ID: 0  Name: George  Balance: $100.00


Part 2.

Write an output operator << to enable writing the object's data members to an output stream. Since you should already have a public method that can be used to write an object's information to an output stream (see part 1 above), you merely need to call it from your output operator.  Output operators take the output stream reference as the first parameter, and a const reference of the class as the second parameter. This operator returns the stream reference, which is what allows these calls to be chained together.

Modify your main function to use the output operator instead of calling the output method directly, to send the object to std::cout.

Example input and output (input is underlined):

Enter the name: George
Enter the balance: 100
Account ID: 0  Name: George  Balance: $100.00
