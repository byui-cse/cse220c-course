C++ and Python: Classes

C++ and Python Class Syntax

This will review how classes work in Python and compare to C++ so that you may translate your knowledge between the two languages.

Defining a Class

A class is defined in C++ much like it is in Python. There are two key differences. First, C++ uses { curly braces } to enclose the contents of a class where Python uses indents. Second, the end of the C++ class definition has a ; semicolon.

PythonC++
class My_Class:
    ''' A simple class. '''

    # One attribute in this class.
    x = 42

    def set(self, x):
        ''' Set the value. '''
        self.x = x

    def get(self):
        ''' Get the value. '''
        return self.x

/*************************
 * A simple class
 *************************/
class MyClass
{
public:
   // Set the value.
   void set(int x)
   {
      this->x = x;
   }

   // Get the value.
   int get()
   {
      return x;
   }

   // One attribute in this class.
   int x;
};

Notice how the comment block describing a class comes before the class definition in C++, where the comment block is inside the class definition in Python.

See also: C++ Reference: Class declaration

Declaring an Object

Remember that an object is one instance of a class. Another way to say this is an object is a type of variable whose datatype is a class. In Python, you can create an object with the assignment operator. In C++, you do the same by declaring a variable that is of the class type. The following example uses the same class declaration as above:

PythonC++
# Declare an object.
mc = My_Class()

# Call the two methods.
mc.set(42);
print(mc.get())

#include <iostream>
using namespace std;

/*************************
 * Main
 *************************/
int main()
{
   // Declare the object.
   MyClass mc;

   // Call the two methods.
   mc.set(42);
   cout << mc.get() << endl;
   return 0;
}

Method Definitions

Methods are just functions associated with a class. In both C++ and Python, method definitions work much like stand-alone function definitions.

PythonC++
class My_Class:
    ''' A simple class. '''

    def simple_function(self):
        ''' No input parameters or
            return values. '''
        print("Hello")

    def one_parameter(self, param):
        ''' One input parameter and
            no return values. '''
        print(param)

    def return_value(self):
        ''' No input parameter but
            one return value. '''
        return 42

/*************************
 * A simple class
 *************************/
class MyClass
{
public:
   // No input parameters or
   //   return values.
   void simple_function()
   {
      cout << "Hello";
   }

   // One input parameter and
   //   no return values.
   void one_parametere(int param)
   {
      cout << param;
   }

   // No input parameter but
   //   one return value.
   int return_value()
   {
      return 42;
   }
};

Note that Python methods always have the self parameter. C++ classes do not, though a similar parameter is implied. It is called this in C++.

See also: C++ Reference: Non-static member functions

Attribute Definitions

In Python, attributes can be added to a class after the class is defined. With C++, attributes are specified at declaration time and the set of attributes cannot change without recompiling the code.

PythonC++
class My_Class:
    ''' A simple class. '''

    # An attribute of unknown type.
    one = None

    # An integer attribute.
    two = 42

    # A list attribute
    three = []

/*************************
 * A simple class
 *************************/
class MyClass
{
public:
   // An attribute of type string.
   // The type of the attribute must 
   // be known at compile time.
   std::string one;

   // An integer attribute. 
   // We can initialize the
   // attribute in a constructor.
   int two;

   // An array attribute.
   // The number of elements
   // must be known at
   // compile time.
   int three[256];
};

Note that we must know the data-type of the attributes when we define the class. If we want an attribute to be initialized, we need to perform the initialization in a constructor.

See also: C++ Reference: Non-static data members

Access Modifiers

C++ allows the author of a class to specify whether a given method (member function) or attribute (member variable) is accessible to the users of the class, or just to the methods of the class itself. Python has a notion of access modifiers, but it is not as strongly enforced as it is with C++.

PythonC++
class my_class:
    ''' A simple class. '''

    def access_to_all(self):
        ''' No underscore means
            all have access. '''
        return 42

    def __access_to_other_methods(self):
        ''' Double-underscore at the 
            beginning suggest the method 
            is private. '''
        return 42

    def _access_to_derived(self):
        ''' Single-underscore at 
            the beginning means 
            protected. '''
        return 42

/*************************
 * A simple class
 *************************/
class MyClass
{
public:
   // Public means all 
   // can access this
   int accessToAll()
   {
      return 42;
   }

private:
   // Private means only
   // methods can access this
   int accessToOtherMethods()
   {
      return 42;
   }

protected:
   // Protected means only
   // methods and derived 
   // classes can access this
   int accessToDerived()
   {
      return 42;
   }
};

With Python, a method or attribute that does not begin with an underscore _ is public. This means the method or attribute is directly accessible from the object. With C++, you use the public keyword to indicate the same.

PythonC++
class My_Class:
    ''' A simple class. '''

    # Public attribute.
    x = 10

    # Public method.
    def get(self):
        return self.x

/****************************
 * A simple class
 ****************************/
class MyClass
{
public:
   // Public attribute.
   int x;

   // Public method.
   int get()
   {
      return x;
   }
};

With Python, a method or attribute that begins with a double-underscore __ is private. This means that the method or attribute is only accessible from other methods in the same class. With C++, you use the private keyword to indicate the same.

PythonC++
class My_Class:
    ''' A simple class. '''

    # Private attribute.
    __x = 10

    # Private method.
    def __get(self):
        return self.__x

/****************************
 * A simple class
 ****************************/
class MyClass
{
private:
   // Private attribute.
   int x;

   // Private method.
   int get()
   {
      return x;
   }
};

With Python, a method or attribute that begins with a single underscore _ is protected. This means that the method or attribute is directly accessible only by methods in the class, or by methods from a derived class. With C++, you use the protected keyword to indicate the same. We will not cover derived classes further in this course, so just know they will be a thing in C++ when using advanced class design techniques.

PythonC++
class My_Class:
    ''' A simple class. '''

    # Protected attribute.
    _x = 10

    # Protected method.
    def _get(self):
        return self._x

/****************************
 * A simple class
 ****************************/
class MyClass
{
protected:
   // Protected attribute.
   int x;

   // Protected method.
   int get()
   {
      return x;
   }
};
